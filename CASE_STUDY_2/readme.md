# Payments Risk Analysis – SQL Case Study

This project analyzes simulated payment transaction data to identify potential risk and fraud patterns.

Key analyses:
- payment behaviour per user
- high decline rate detection
- transaction velocity within 24 hours
- multi-device and multi-country activity
- merchant concentration patterns
- rapid merchant switching

Techniques used:
- SQL aggregations
- CTE
- window functions
- time-window analysis
- self joins

Tools:
- SQLite
- DBeaver
- Power BI
